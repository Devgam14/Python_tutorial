// system architecture

Telecom App
Flow Documentation

PyDAL · SQL · Terminal Interface _
CORE SYSTEM
USER FLOW
ADMIN FLOW
DECISION POINT
EXIT
STEP 01
Application Boot
  • User runs the application from the terminal
  • main.py starts execution
  • PyDAL initializes SQL database connection
  • Required tables are loaded or created if absent
  • Welcome banner rendered to stdout
STEP 02
Authentication Decision
  • System displays main menu: Login / Register / Exit
  • User selects an option via terminal prompt
◆ DECISION POINT — Select route
▸ LOGIN → Step 3B
▸ REGISTER → Step 3A
✕ EXIT → Step 10
// parallel auth paths
STEP 03A
User Registration
  • Enter phone number
  • System validates phone format
  • User sets PIN / password
  • System generates unique user ID
  • Record saved via PyDAL → SQL
  • Confirmation message displayed
  • Redirected to Login (Step 3B)
STEP 03B
User Login
  • Enter phone number and PIN
  • Credentials validated against database
  • Account status checked (active / blocked)
◆ DECISION POINT
▸ VALID → Step 4
✕ INVALID → Error → Retry
STEP 04
Role-Based Routing
  • System reads role field from authenticated user record
  • Routes to the appropriate menu interface
◆ DECISION POINT — Role check
▸ USER → Steps 5–8
▸ ADMIN → Step 9
// role-based flow split
STEP 05
User Menu
  • Check balance
  • Recharge airtime
  • Buy data
  • Make call
  • Send SMS
  • View usage history
  • Logout
STEP 06
Buy Data Flow
  • User selects "Buy Data"
  • System fetches current data pricing
  • Available bundles displayed
  • User selects bundle
  • Balance validation check
◆ DECISION POINT
▸ SUFFICIENT → Deduct → Log → Success
✕ INSUFFICIENT → Error
STEP 07
Call Simulation
  • User enters destination number
  • System generates random call duration
  • Progress simulated via time.sleep()
  • Cost calculated from duration
  • Balance updated in database
  • Call log persisted to SQL
STEP 08
View History
  • Transactions fetched via PyDAL query
  • Data formatted with PrettyTable
  • History rendered in terminal
  • User returned to menu
STEP 09
Admin Menu
  • View all users
  • View system stats
  • Reset user balance
  • Block / Unblock user
  • Logout
PyDAL executes privileged DB operations · Results rendered · Admin returned to menu
STEP 10
Logout / Exit
  • User logs out or selects Exit
  • PyDAL database session closed
  • Program terminates safely
